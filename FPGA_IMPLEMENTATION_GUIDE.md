# TinyMobileNet-XS FPGA Inference Implementation
## Nexys A7-100T DDR4 (xc7a100tcsg324-1)

---

## 📋 Project Overview

This directory contains the **complete FPGA implementation** of the TinyMobileNet-XS neural network for pedestrian crossing-intention estimation on Xilinx Artix-7 FPGA.

**Key Features:**
- ✅ Pipelined inference architecture
- ✅ INT8 weight quantization support
- ✅ UART interface for host communication
- ✅ LED output for real-time predictions
- ✅ Pre-loaded test vectors in BRAM
- ✅ Python host interface for verification
- ✅ Vivado 2024.1 compatible

## 🚀 Quick Start

### 1. **Generate Test Vectors** (Python)

```bash
python generate_fpga_test_vectors.py
```

This will:
- Load trained PyTorch model
- Generate INT8 test samples
- Export to Verilog format
- Report FP32 vs INT8 accuracy

### 2. **Create Vivado Project**

```bash
vivado -mode batch -source fpga/create_project.tcl
```

Or manually in Vivado:
```tcl
cd fpga
open_project vivado_project/tinymobilenet_fpga.xpr
```

### 3. **Synthesize & Implement**

In Vivado TCL:
```tcl
launch_runs synth_1 -jobs 4
wait_on_run synth_1

launch_runs impl_1 -jobs 4
wait_on_run impl_1

launch_runs impl_1 -to_step write_bitstream
```

### 4. **Program FPGA**

```tcl
open_hw_manager
connect_hw_server
open_hw_target
current_hw_device [get_hw_devices xc7a100t_0]
refresh_hw_device -update_hw_probes false [current_hw_device]
set_property PROGRAM.FILE {fpga/vivado_project/tinymobilenet_fpga.runs/impl_1/tinymobilenet_top.bit} [current_hw_device]
program_hw_devices [current_hw_device]
```

### 5. **Test with Python Host**

**Option A: Offline verification (no FPGA needed)**
```bash
python fpga_host_interface.py --offline
```

**Option B: Live FPGA testing**
```bash
python fpga_host_interface.py --port COM3 --num-samples 5
```

## 🔌 Hardware Connections (Nexys A7-100T)

### **Input Signals:**
| Signal | Pin | Purpose |
|--------|-----|---------|
| `clk` (100 MHz) | E3 | System clock |
| `rst_n` (button) | D9 | Active-low reset |
| `start_button` (BTN1) | C9 | Inference trigger |
| `uart_rx` | A9 | Serial RX from PC |

### **Output Signals:**
| Signal | Pins | Purpose |
|--------|------|---------|
| `class_led[1]` (Green) | H17 | Crossing prediction |
| `class_led[0]` (Red) | K15 | Not-crossing prediction |
| `confidence_led[7:0]` | J15,J13,H13,G13,B11,A11,D12,D11 | Confidence score (8-bit) |
| `state_led[3:0]` | E15,F15,G15,H15 | FSM state (debug) |
| `inference_done` | E14 | Inference complete flag |
| `uart_tx` | B10 | Serial TX to PC |

## 📊 Inference Architecture

### **Data Flow:**
```
Input Data (49,152 bytes)
    ↓
Conv1 (64×64×3 → 32×32×8)
    ↓
MobileBottleneck A,B,C (downsampling)
    ↓
Temporal Shift Module (TSM - zero-cost)
    ↓
Conv_pw (→ 48 channels)
    ↓
Global Average Pool (8×8×48 → 48)
    ↓
FC1: 48 → 32 (ReLU)
    ↓
FC2: 32 → 2 (logits)
    ↓
Softmax + Argmax (class + confidence)
    ↓
Output LEDs + UART
```

### **Latency Breakdown:**
| Layer | Cycles | Latency (@ 100MHz) |
|-------|--------|-------------------|
| Conv1 | ~1,024 | 10.24 µs |
| Bottleneck layers | ~4,096 | 40.96 µs |
| TSM | ~100 | 1.0 µs |
| Conv_pw | ~500 | 5.0 µs |
| GAP | ~50 | 0.5 µs |
| FC layers | ~100 | 1.0 µs |
| **Total** | **~6,000** | **~60 µs** |

**Note:** Actual latency will be higher (~1-10 ms) due to memory stalls and control overhead.

## 💾 Memory Layout

### **BRAM (Block RAM):**
- **Weights:** ~7.2 KB (INT8 quantized)
- **First test sample:** ~49 KB
- **Total used:** ~56 KB / 270 available blocks ✅ (Very efficient!)

### **DDR3 (External):**
- Optional: Store additional test samples (16 samples = 784 KB)
- Currently not used in basic implementation

## 🔬 Simulation in Vivado

### **Run RTL Simulation:**

1. In Vivado:
   ```tcl
   set_property SOURCE_SET sources_1 [current_project]
   open_run synth_1 -name synth_1
   launch_simulation
   ```

2. In simulation, press the button:
   ```tcl
   run 100 us
   set_property value 1 [get_hw_probes start_button -of_objects [current_hw_device]]
   run 100 us
   set_property value 0 [get_hw_probes start_button -of_objects [current_hw_device]]
   run 1 ms
   ```

3. View waveforms in `tinymobilenet_sim.vcd`

## 🔧 Customization & Tuning

### **Increase Inference Accuracy:**
1. Use Quantization-Aware Training (QAT) in `quantization_proper.py`
2. Retrain with `python train.py --qat`
3. Re-export weights

### **Reduce Latency:**
1. Increase pipeline depth (trade memory for speed)
2. Use more DSPs for parallel operations
3. Implement systolic array for convolutions

### **Increase Throughput:**
1. Pipeline multiple inferences
2. Use DDR3 for weight caching
3. Implement streaming architecture

## 🐛 Troubleshooting

### **Problem: FPGA doesn't respond to button**
- ✓ Check button is wired to correct pin (C9)
- ✓ Verify reset button was pressed first (D9)
- ✓ Check LEDs light up (test basic I/O)

### **Problem: UART communication fails**
- ✓ Use correct COM port: `python fpga_host_interface.py --port COM3`
- ✓ Verify baud rate matches: 115200
- ✓ Check USB cable is connected
- ✓ Check device manager for serial port

### **Problem: Synthesis/Implementation fails**
- ✓ Update Vivado to 2024.1
- ✓ Check that `xc7a100tcsg324-1` is selected
- ✓ Verify all .v files are in rtl/ directory
- ✓ Check constraints file syntax
