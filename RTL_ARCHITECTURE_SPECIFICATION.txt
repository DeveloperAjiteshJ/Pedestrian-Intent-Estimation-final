INFERENCE PIPELINE (Frame-by-Frame RTL Flow)
=============================================

Step 1: INPUT STAGE (64×64×3 RGB frame)
  ┌─────────────────────────────┐
  │ Frame from DDR3             │ (1920×1080 downsampled to 64×64)
  │ Or USB/PCIe input stream    │
  └──────────┬──────────────────┘
             │
             ▼
  ┌─────────────────────────────────────────────────────┐
  │ INPUT QUANTIZATION & BUFFERING                      │
  │ • Convert RGB to UINT8 (if not already)             │
  │ • Store in input BRAM buffer (12 KB for 64×64×3)   │
  │ • Index management: frame_count mod T               │
  └──────────┬──────────────────────────────────────────┘
             │
  
Step 2: CONV PIPELINE (Per-Frame Processing)
  Layer 0: Conv1 (3×3, stride=2) — 3→8 channels
    Input:  64×64×3
    Output: 32×32×8
    ┌──────────────────────────┐
    │ Line Buffer (3×64×3)     │ ← 0.6 KB BRAM
    │ Kernel Cache (3×3×3×8)   │ ← 0.2 KB BRAM
    │ Systolic Array (8 parallel)
    │ Accumulator (16-bit)     │
    └──────────┬───────────────┘
               ▼
  
  Layer 1: Bottleneck A (DW 3×3, stride=1)
    DW Conv: 8 channels → 8 channels (32×32)
    PW Conv: 32×32×8 → 32×32×8
    ┌──────────────────────────┐
    │ Depthwise MAC Array      │ ← 8 DSPs
    │ Pointwise MAC Array      │ ← 8 DSPs
    └──────────┬───────────────┘
               ▼
  
  Layer 2: Bottleneck B (stride=2 reduction)
    Expand: 1×1, 8→32 channels
    DW:     3×3, stride=2, 32 channels
    Project: 1×1, 32→12 channels
    ┌──────────────────────────┐
    │ MAC Array (32 DSPs)      │
    │ Output: 16×16×12         │
    └──────────┬───────────────┘
               ▼
  
  Layer 3: Bottleneck C ×2 (with TSM insertion)
    Block 1: Stride=2 reduction (12→16 channels)
    Block 2: Stride=1 (16 channels)
    ┌──────────────────────────┐
    │ MAC Array (48 DSPs)      │
    │ Output: 8×8×16           │
    │                          │
    │ ✨ TSM BUFFER            │
    │    (circular BRAM)       │
    │    4×(8×8×16) = 4 KB    │
    └──────────┬───────────────┘
               ▼
  
  Layer 4: Conv_pw (1×1 expansion)
    16→48 channels, 8×8 spatial
    ┌──────────────────────────┐
    │ Pointwise MAC Array      │ ← 16 DSPs
    └──────────┬───────────────┘
               ▼

Step 3: TEMPORAL FUSION
  ┌──────────────────────────────────┐
  │ Global Average Pool              │
  │ 8×8×48 → 1×1×48 (48-dim vector)│
  │ (Simple: accumulate & divide)    │
  └──────────┬───────────────────────┘
             │
  ┌──────────▼───────────────────────┐
  │ TSM Temporal Average (T frames)  │
  │ Average T pooled vectors         │
  │ Result: 48-dim aggregated vector │
  └──────────┬───────────────────────┘
             │

Step 4: CLASSIFICATION HEAD
  ┌──────────────────────────┐
  │ FC1: 48→32 (ReLU)        │ ← 2 KB BRAM (weights)
  │ FC2: 32→2 (logits)       │ ← 0.5 KB BRAM
  └──────────┬───────────────┘
             │
  ┌──────────▼───────────────────────┐
  │ Output: 2 logits                 │
  │ [logit_not_crossing,             │
  │  logit_crossing]                 │
  └──────────┬───────────────────────┘
             │
  ┌──────────▼───────────────────────┐
  │ Softmax (in firmware/Microblaze) │
  │ Convert to probabilities         │
  │ Output: {class, confidence}      │
  └──────────────────────────────────┘

1. INPUT_BUFFER (12 KB BRAM-based)
────────────────────────────────────

module input_buffer #(
  parameter FRAME_H = 64,
  parameter FRAME_W = 64,
  parameter FRAME_C = 3,
  parameter BRAM_WIDTH = 72,  // AXI protocol
  parameter BRAM_DEPTH = 2048
) (
  input  clk, rst,
  
  // AXI slave (write from DDR/USB)
  input  [BRAM_WIDTH-1:0] s_axi_wdata,
  input  [8:0] s_axi_waddr,
  input  s_axi_wvalid,
  output s_axi_wready,
  
  // Output stream to Conv pipeline
  output [7:0] frame_data_out,   // UINT8 pixel
  output frame_valid,
  input  frame_ready,
  
  // Control
  input  [2:0] buffer_select  // Select which frame (0 to T-1)
);
  // Implementation:
  // - Dual-port BRAM (one AXI write, one stream read)
  // - Address generation: buffer_select, row, col, channel
  // - Pipelining: register output for timing
endmodule


2. LINE_BUFFER (Per-Conv Layer)
────────────────────────────────

module line_buffer #(
  parameter WIDTH = 64,
  parameter CHANNELS = 3,
  parameter KERNEL_ROWS = 3,
  parameter BRAM_WIDTH = 72
) (
  input  clk, rst,
  
  input  [7:0] pixel_in,           // Current pixel
  input  pixel_valid,
  output [7:0] window[0:2][0:2],  // 3×3 window
  output window_valid
);
  // Implementation:
  // - Three 64×CHANNELS line buffers in BRAM
  // - Shift registers for 3×3 window extraction
  // - Output: window with spatial & channel dimensions
endmodule


3. CONV_KERNEL_SYSTOLIC (Depthwise/Pointwise)
──────────────────────────────────────────────

module conv_kernel_systolic #(
  parameter KERNEL_H = 3,
  parameter KERNEL_W = 3,
  parameter IN_CHANNELS = 8,
  parameter OUT_CHANNELS = 8,
  parameter DATA_WIDTH = 8,
  parameter WEIGHT_WIDTH = 8,
  parameter ACCUM_WIDTH = 32
) (
  input  clk, rst,
  
  // Input window (3×3×IN_C)
  input  [DATA_WIDTH-1:0] window[0:2][0:2][0:IN_CHANNELS-1],
  input  window_valid,
  
  // Weights from BRAM
  input  [WEIGHT_WIDTH-1:0] weights[0:KERNEL_H*KERNEL_W*IN_CHANNELS*OUT_CHANNELS-1],
  input  [WEIGHT_WIDTH-1:0] bias[0:OUT_CHANNELS-1],
  
  // Output
  output [7:0] output_data[0:OUT_CHANNELS-1],  // Quantized UINT8
  output output_valid,
  
  // Quantization scale (from software/calibration)
  input  [15:0] scale[0:OUT_CHANNELS-1]
);
  // Implementation:
  // - Systolic array: 3×3 multiplier array
  // - Each of OUT_CHANNELS computes one output channel
  // - Pipelined multiply-accumulate (3 cycles for 3×3 kernel)
  // - Final quantization: (accum * scale) >> 8, clip [0, 255]
  
  // Pseudo-RTL:
  // For each output channel oc:
  //   accum[oc] = bias[oc];
  //   for ky = 0 to 2:
  //     for kx = 0 to 2:
  //       for ic = 0 to IN_CHANNELS-1:
  //         accum[oc] += window[ky][kx][ic] * weights[...];
  //   output_data[oc] = (accum[oc] * scale[oc]) >> 8;
endmodule


4. GLOBAL_AVG_POOL (Spatial Reduction)
──────────────────────────────────────

module global_avg_pool #(
  parameter H = 8,
  parameter W = 8,
  parameter CHANNELS = 48,
  parameter ACCUM_WIDTH = 32
) (
  input  clk, rst,
  
  input  [7:0] feature_in,    // Stream input
  input  feature_valid,
  
  output [15:0] feature_out,  // Accumulated 16-bit value
  output feature_out_valid
);
  // Implementation:
  // - Accumulator array [0:CHANNELS-1]
  // - For each pixel in 8×8 spatial, accumulate across all CHANNELS
  // - Divide by (H*W) = 64 when complete
  // - Output: 16-bit per channel (before quantization)
endmodule


5. TSM_BUFFER (Temporal Shift Module — Circular BRAM)
───────────────────────────────────────────────────────

module tsm_buffer #(
  parameter FRAME_H = 8,
  parameter FRAME_W = 8,
  parameter CHANNELS = 16,
  parameter MAX_T = 4,
  parameter BRAM_WIDTH = 72
) (
  input  clk, rst,
  
  // Write port (from Layer 3 bottleneck output)
  input  [7:0] feat_in,
  input  feat_write_valid,
  input  [2:0] frame_idx,      // Current frame (0 to T-1)
  
  // Read port (for temporal fusion & next frame)
  output [7:0] feat_out,
  output feat_read_valid,
  input  feat_read_ready,
  input  [2:0] frame_read_idx,
  
  // Shift control (which channels to shift)
  input  [3:0] shift_channels  // e.g., shift channels 0-3 out of 16
);
  // Implementation:
  // - BRAM storage: T × H × W × C = 4 × 8 × 8 × 16 = 4,096 bytes
  // - Circular addressing: addr = (frame_idx % T) * (H*W*C) + ...
  // - Shift implemented via read pointer offset (no data movement)
  // - For shifted channels: read from (frame_idx - 1 + T) % T
  
  // Address mapping:
  // addr[15:0] = {frame_idx[1:0], h[2:0], w[2:0], c[3:0], byte[1:0]}
endmodule


6. FC_LAYER (Fully Connected — Small & Simple)
────────────────────────────────────────────────

module fc_layer #(
  parameter IN_DIM = 48,
  parameter OUT_DIM = 32,
  parameter DATA_WIDTH = 8,
  parameter WEIGHT_WIDTH = 8,
  parameter ACCUM_WIDTH = 32
) (
  input  clk, rst,
  
  input  [DATA_WIDTH-1:0] input_vec[0:IN_DIM-1],
  input  input_valid,
  
  // Weights: stored in BRAM [IN_DIM × OUT_DIM]
  input  [WEIGHT_WIDTH-1:0] weights[0:IN_DIM*OUT_DIM-1],
  input  [WEIGHT_WIDTH-1:0] bias[0:OUT_DIM-1],
  
  output [7:0] output_vec[0:OUT_DIM-1],
  output output_valid,
  
  input  [15:0] scale[0:OUT_DIM-1]  // Quantization scale
);
  // Implementation:
  // - Simple systolic: IN_DIM multipliers in parallel
  // - One output per cycle (fully pipelined)
  // - Pseudo-RTL:
  //   for oo = 0 to OUT_DIM-1:
  //     accum = bias[oo];
  //     for ii = 0 to IN_DIM-1:
  //       accum += input_vec[ii] * weights[ii * OUT_DIM + oo];
  //     output_vec[oo] = (accum * scale[oo]) >> 8;
endmodule


7. CONTROL_FSM (Main Inference Controller)
──────────────────────────────────────────

module control_fsm (
  input  clk, rst,
  
  // Signals from/to Microblaze
  input  [31:0] frame_count,    // How many frames to process
  input  start_inference,
  output done_inference,
  
  // Control signals to pipeline stages
  output [1:0] stage_enable,    // Enable each pipeline stage
  output [2:0] frame_idx,       // Current frame index (0 to T-1)
  output clear_accumulators,    // For GAP, FC
  
  // Status
  output [7:0] pipeline_state   // Debug: which stage active
);
  // FSM states:
  // 0: IDLE
  // 1: LOAD_INPUT (wait for frame_count frames in buffer)
  // 2: PROCESS_PIPELINE (run Conv layers)
  // 3: TEMPORAL_FUSION (average across T frames)
  // 4: FC_INFERENCE (run FC1, FC2)
  // 5: DONE (write logits to output register)
  
  // Pseudo-RTL:
  // case (state)
  //   IDLE:
  //     if (start_inference) next_state = LOAD_INPUT;
  //   LOAD_INPUT:
  //     stage_enable[0] = 1;  // Input buffer streaming
  //     if (frames_loaded == frame_count) next_state = PROCESS_PIPELINE;
  //   PROCESS_PIPELINE:
  //     stage_enable[1:0] = 2'b11;  // Conv pipeline
  //     if (pipeline_done) next_state = TEMPORAL_FUSION;
  //   ... and so on
endmodule

PART 4: DATAFLOW & PIPELINING STRATEGY
================================================================================

PIPELINE STAGES (7 stages total):
  Stage 0: Input quantization & buffering (latency: ~100 clks)
  Stage 1: Conv1 + Bottleneck A (latency: ~2k clks)
  Stage 2: Bottleneck B (latency: ~0.5k clks)
  Stage 3: Bottleneck C ×2 + Conv_pw (latency: ~1k clks)
  Stage 4: Global Average Pool (latency: ~5k clks for 48 channels)
  Stage 5: Temporal fusion (TSM average) (latency: ~200 clks)
  Stage 6: FC1 + FC2 (latency: ~100 clks)
  
Total latency per frame:
  ~1.29M MACs ÷ 64 DSPs (assuming 32-bit per cycle) ≈ 20k cycles at 100 MHz
  20k cycles ÷ 100 MHz = 200 μs per frame
  × 4 frames = 800 μs total (1.25 kFPS) ← Conservative estimate
  
Actual throughput with pipelining:
  - Each stage processes one frame at a time
  - With full pipeline saturation: 1 output per 200 μs = 5 kFPS
  - More realistic (with stalls): 2-3 kFPS


MEMORY ACCESS PATTERNS:
  
  Read Bandwidth:
    Conv1: 64×64×3 = 12,288 bytes/frame
    Line buffers: 3×64 = 192 bytes/frame per layer
    Weights (cached): negligible per-frame (reused)
    
  Write Bandwidth:
    Conv outputs: variable size (32×32×8 → 8×8×16 → logits)
    TSM buffer: 8×8×16 = 1,024 bytes per frame
    
  Total: ~20 KB/frame, easily sustainable from local BRAM

PART 6: I/O INTERFACES & PROTOCOL SPECIFICATIONS
================================================================================

AXI4 SLAVE (Input frames from DDR3):
  Width: 72-bit (64-bit data + control)
  Burst length: Configurable (typically 16-32)
  Address: 32-bit
  Flow: Continuous streaming into input buffer

AXI4 MASTER (Output logits to DDR3):
  Width: 32-bit (2 logits × 16-bit each)
  Write: Single transaction per inference completion

AXI-STREAM (Optional: Real-time video input)
  Width: 24-bit (8-bit RGB × 3 channels)
  Signals: tdata, tvalid, tready, tlast
  Use case: Direct video stream without DDR

Register Interface (Microblaze control):
  Base address: 0x4000_0000
  
  Offset 0x00: CONTROL Register
    [0]: start_inference (write 1 to start)
    [1]: interrupt_enable
    [7:4]: frame_count (0-15 frames)
  
  Offset 0x04: STATUS Register
    [0]: done (read-only)
    [1]: error
    [7:4]: pipeline_state
  
  Offset 0x08: OUTPUT_LOGITS[0] (16-bit)
    Logit for class 0 (not crossing)
  
  Offset 0x0A: OUTPUT_LOGITS[1] (16-bit)
    Logit for class 1 (crossing)
  
  Offset 0x0C: LATENCY_CYCLES (32-bit)
    Cycle count for inference (debugging)


================================================================================
PART 7: QUANTIZATION & NUMERICAL PRECISION
================================================================================

INT8 QUANTIZED WEIGHTS & ACTIVATIONS:
  
  Range: [-128, 127] for weights
  Range: [0, 255] for activations (UINT8)
  
  Quantization formula:
    q_val = round(float_val × scale / 128)
    clip(q_val, -128, 127)  // For weights
    clip(q_val, 0, 255)     // For activations
  
  Scale factors (stored per-layer, float32 or fixed):
    Typically in range [0.01, 10.0]
    Determined during post-training calibration
  
  Dequantization (in hardware):
    accum = (int16) weight × (int16) activation
    dequant = accum × scale / 128  (can be pre-computed as (scale << 8) / 128)

ACCUMULATOR PRECISION:
  Intermediate: 16-bit signed (avoids overflow for 8×8 multiply)
  Final accumulator: 32-bit signed (handles 3×3 kernel × 8 channels sums)
  Saturating arithmetic: Clamp to [-128, 127] after quantization
